### 1️⃣ API의 개념

- API는 미리 구현된 기능을 외부 사용자가 활용할 수 있도록 제공하는 인터페이스입니다.
- 증권사 API를 통해 주가 데이터 조회, 주문 등 다양한 기능을 사용할 수 있습니다.
    - 정해진 규칙에 따라 요청을 보내면 원하는 작업의 결과를 응답으로 받을 수 있습니다.
    - 직접 구현하기 어려운 기능도 API를 통해 쉽게 활용할 수 있습니다.

### 2️⃣ 한국투자증권 API 시작하기

- 한국투자증권은 오픈API를 통해 주식 거래 관련 기능을 제공합니다.
- API 사용을 위해서는 회원가입과 사용 등록이 필요합니다.
    - 실전투자와 모의투자 중 선택할 수 있습니다.
    - 모의투자는 실제 자금 없이 거래를 연습할 수 있는 계좌입니다.
- API 키와 시크릿을 발급받아야 API를 사용할 수 있습니다.

<aside>
⚠️ **여기서 잠깐!**

- API 키와 시크릿은 계좌 접근 권한을 가진 중요한 보안 정보입니다.
- 악의적인 사용자가 이 정보를 획득하면 무단으로 자금을 인출할 수 있습니다.
- 대량의 의도치 않은 주문으로 금전적 손실이 발생할 수 있습니다.
- API 키와 시크릿은 절대 외부에 노출되지 않도록 안전하게 관리해야 합니다.
</aside>

### 3️⃣ API 문서 활용하기

- API 문서는 제공되는 기능과 사용 방법을 설명하는 안내서입니다.
- API 문서에는 사용 가능한 기능과 요청 형식이 상세히 정리되어 있습니다.
    - 어떤 기능을 사용할 수 있는지 확인할 수 있습니다.
    - 각 기능의 구체적인 사용 방법을 파악할 수 있습니다.
- 모든 Open API는 문서를 제공하므로 이를 참고해 필요한 기능을 활용할 수 있습니다.

<aside>
⚠️ **여기서 잠깐!**

- Open API(공개 API)는 누구나 사용할 수 있도록 공개된 API로, 대부분 상세한 문서를 함께 제공합니다.
- 비공개 API는 특정 조직이나 파트너만 접근할 수 있으며, 문서 접근도 제한적일 수 있습니다.
- Open API는 보통 개발자 포털을 통해 API 키 발급, 사용량 모니터링 등의 기능도 함께 제공합니다.
</aside>

<aside>
➕  **API 문서의 주요 구성 요소**

- Endpoint: API 요청을 보낼 주소를 설명합니다.
- Parameters: API 요청 시 필요한 매개변수를 설명합니다.
- Request Example: API 요청의 예시를 보여줍니다.
- Response Example: API 응답의 예시를 보여줍니다.
- Error Codes: 발생 가능한 오류와 대처 방법을 설명합니다..
</aside>
